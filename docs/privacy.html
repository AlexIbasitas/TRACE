Privacy Policy for TriageMate

Last updated: July 20, 2025

TriageMate is an IntelliJ Platform plugin that provides AI-powered debugging assistance for test failures. When tests fail, the plugin analyzes the failure data and project structure to generate intelligent suggestions for fixing the issues. The plugin uses AI services to process test failures and project information, while storing API keys and chat history locally on your machine.

1. Information We Collect

Test Failure Data

  • What: Stack traces, error messages, and test failure details
  • Why: To generate AI-powered debugging suggestions
  • How: Sent to AI services (OpenAI API) for analysis
  • Storage: Not permanently stored by us

Project Structure Information

  • What: Project file structure and step definition locations
  • Why: To locate and analyze step definition implementations
  • How: Parsed locally and sent to AI services for context
  • Storage: Not permanently stored by us

API Keys

  • What: Your OpenAI API key or other AI service credentials
  • Why: To authenticate with AI services for analysis
  • How: Stored locally on your machine
  • Storage: Local storage only, never sent to our servers

Chat History

  • What: Conversation history with AI debugging suggestions
  • Why: To provide conversation continuity and context
  • How: Stored locally on your machine
  • Storage: Local storage only, never sent to our servers

2. How We Use Your Information

  • Generate AI-powered debugging suggestions
  • Improve plugin functionality and user experience
  • Provide technical support
  • Analyze usage patterns to improve the plugin

3. AI Predictive Analytics

TriageMate uses AI predictive analytics to analyze test failures and generate debugging suggestions. This involves:

  • Data Processing: Test failure data and project structure information are processed by AI services
  • Predictive Analysis: AI algorithms analyze patterns to predict potential solutions
  • Intelligent Recommendations: AI generates specific debugging suggestions based on the analysis
  • Continuous Learning: AI services may improve their suggestions based on patterns in the data

4. Third-Party Services

We use the following third-party services:

  • OpenAI API: For AI-powered analysis and debugging suggestions

When you enable AI features, test failure data is sent to OpenAI for analysis. Data is not permanently stored by OpenAI (subject to their privacy policy).

5. Data Storage and Security

  • Local Storage: API keys and chat history stored locally on your machine
  • Temporary Processing: Test data sent to AI services for analysis, not permanently stored
  • No Permanent Cloud Storage: We do not permanently store your data on our servers

6. Your Rights (GDPR/CCPA)

You have the right to:

  • Access your personal data
  • Request deletion of your data
  • Withdraw consent at any time
  • Opt out of data collection

7. Data Retention

  • Local data: Stored until you uninstall the plugin or clear settings
  • AI service data: Not permanently stored by us
  • Chat history: Stored locally until manually cleared
  • API Keys: Stored locally until removed from settings

8. Legal Basis (GDPR)

  • Consent: You explicitly consent to data processing when enabling AI features
  • Legitimate Interest: Providing debugging suggestions to improve your development experience
  • Contract: Processing necessary to provide the plugin functionality

9. International Data Transfers

Data may be transferred to AI services located outside your country. We ensure appropriate safeguards are in place for such transfers.

10. Children's Privacy

This plugin is designed for professional developers and does not knowingly collect data from children under 13.

11. Changes to This Policy

We may update this policy. We will notify you of significant changes via the plugin or GitHub repository.
